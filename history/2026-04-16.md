# 2026-04-16

## Сессия: Анализ деградации `The Sunny Morning` на `nllb33` и фикс anti-repeat decode path

### Что выявлено

- книга `The Sunny Morning.txt` в базе `book_89842f038a1a` была импортирована через `facebook/nllb-200-3.3B`
- сегментация книги оказалась рабочей:
  - заголовки, chapter heading, предложения и реплики были выделены в отдельные segment-ы без потери порядка
- корневая деградация шла из translation layer:
  - `The Sunny Morning` -> повтор `Sunny Morning ...`
  - `Chapter 1: The New Day` -> `Глава 1: The New Day ...` с повтором title-хвоста
  - `"Good morning, Luna!" Tom says.` -> `"До". "До". ...`
  - `The cat says, "Meow."` -> шум/повтор `кококо ...`

### Что изменено

- обновлён [engine/translator.py](/mnt/d/Programs/LEXO/engine/translator.py)
  - `Nllb33Provider` отделён от общего decode path
  - для `nllb33` добавлены отдельные decode profiles:
    - base
    - retry
  - `nllb33` переведён на single-segment decode вместо общего batched path
  - добавлен dynamic `max_decoding_length` по длине segment-а
  - добавлен technical guard `_is_degraded_translation(...)` против loop/repetition output
  - при деградации для `nllb33` включается повторный более жёсткий decode pass

### Проверка

- `python3 -m py_compile engine/translator.py` -> OK
- helper smoke-check:
  - `The Sunny Morning` + looped output -> degraded = `True`
  - `Chapter 4: Home` + repeated `Home` output -> degraded = `True`
  - `The sun is bright.` + `Солнце ярко светит.` -> degraded = `False`
  - `It is a very good day.` + `Это очень хороший день.` -> degraded = `False`
- прямой runtime smoke-test `Nllb33Provider()` из WSL не выполнен:
  - в текущем Linux Python отсутствует пакет `ctranslate2`
  - живую модель нужно перепроверить из проектной среды / launcher-а, где доступен CT2 runtime

### Итог

- исправление внесено в translation layer, а не в segmenter/storage
- цель фикса: остановить повтор/раздувание коротких `nllb33` segment-ов без возврата к book-specific hardcode
