ТЗ: переделка экрана чтения под верхнюю строку перевода по нажатию на слово
Цель

Переделать текущий экран чтения книги из режима paragraph cards в режим reader-first, где:

основной контент — английский текст книги
русский перевод не показывается постоянно
сверху есть один фиксированный блок перевода
при нажатии на английское слово верхний блок обновляется и показывает:
перевод выбранного слова
контекст вокруг него в пределах доступной ширины блока
1. Что должно быть на выходе

Экран чтения должен состоять из 3 частей:

1. Верхний блок перевода

Фиксированная панель сверху экрана

Назначение:

показывать перевод по нажатию на английское слово
отображать один текстовый блок
в центре логически находится перевод выбранного слова
по краям отображается соседний контекст, насколько влезает в блок

Особенности:

блок не скроллится отдельно
блок не анимируется как бегущая строка
блок не содержит английский текст
блок обновляется сразу при нажатии на слово
до первого нажатия может показывать:
пустое состояние
либо короткую подсказку, например: Tap a word to see translation
2. Основная область чтения

Отображает книгу как поток текста, а не как карточки

Назначение:

служить основным reader view
показывать только английский текст
поддерживать нажатие на отдельные слова

Особенности:

абзацы допустимы как логические блоки текста
но без Card
без § номера
без SRC/TGT меток
без постоянного targetText под английским текстом
3. TTS-панель

Оставить существующую TTS-панель

Особенности:

не ломать существующую интеграцию с TTS
допустимо оставить текущее расположение сбоку на этом этапе
если потребуется, разрешается только визуальная подстройка отступов и размеров, без переписывания TTS-логики

3. Новая UX-модель
Основной принцип

Пользователь читает только английский текст

Русский перевод показывается только по нажатию на слово

Механика

При нажатии на английское слово:

определяется выбранное слово
определяется перевод для этого слова
формируется строка для верхнего блока:
выбранный перевод
немного соседнего контекста слева
немного соседнего контекста справа
верхний блок обновляется
выбранное английское слово в тексте визуально подсвечивается

9. Требования к обработке нажатия на слово
Нужно реализовать

При нажатии на слово UI должен получить минимум такую информацию:

индекс параграфа

На основе этого нужно построить строку верхнего блока
MVP-логика допустима упрощённая

На этом этапе разрешается использовать простую схему:

брать targetText текущего абзаца

отображать результат в верхней строке
Важно

Даже если соответствие пока не идеальное, механика должна уже работать визуально

То есть сейчас нужен рабочий UX-скелет, а не идеальный linguistic alignment

11. Требования к позиции чтения
Сейчас

17. Что считается успешным результатом

Результат считается принятым, если одновременно выполняется следующее:

экран больше не показывает paragraph cards
английский текст читается как основной контент
русский перевод не висит постоянно под каждым абзацем
сверху есть отдельный фиксированный блок перевода
по нажатию на английское слово верхний блок обновляется
выбранное слово визуально выделяется
TTS-панель не сломана
существующая загрузка книги работает
текущая позиция чтения по paragraph index не сломана

20. Формулировка задачи для исполнения

Можно дать агенту в таком виде:

Переделать ReaderScreen из paragraph-card view в reader-first view.
Добавить фиксированный верхний TranslationContextBar, который обновляется по нажатию на английское слово в тексте.
Основной reader area должен показывать только английский текст как поток чтения, без карточек, без номеров параграфов, без source/target labels и без постоянного русского перевода.
Сделать интерактивный рендер абзацев с возможностью нажимать на отдельные слова, подсвечивать выбранное слово и обновлять верхний блок перевода.
Backend и текущие API не ломать. Позицию чтения временно продолжать хранить по paragraph_index.
Логику формирования текста верхнего блока вынести отдельно в helper.
