## Сессия: Точечный фикс reporting-verb alignment и cleanup явных EN->RU ошибок

### Контекст

- пользователь попросил диагностировать качество перевода и объяснить, почему часть слов остаётся без перевода
- по живой диагностике активной книги подтверждено:
  - `says / asks / whispers / thinks` часто теряли target span при русском обратном порядке `говорит Том / спросил Том / шепчет Том / думает Том`
  - в segment translation оставались явные дефекты уровня `Доброй вечер`, `Мев`, `серыя`, `делает завтрак`, `синий футболка`

### Что изменено

- обновлён [engine/word_alignment.py](/mnt/d/Programs/LEXO/engine/word_alignment.py)
  - добавлен `REPORTING_VERB_TRANSLATIONS` с candidate-формами для:
    - `say/says`
    - `think/thinks`
    - `ask/asks`
    - `whisper/whispers`
    - `reply/replies`
    - `answer/answers`
  - в `build_word_mappings(...)` добавлен post-pass `_rescue_reporting_verb_targets(...)`
  - новый rescue-pass:
    - ищет reporting verb candidate не только по текущей позиции
    - умеет подбирать target token рядом с anchor-именем
    - спасает кейсы с обратным порядком слов в русском segment

- обновлён [engine/translator.py](/mnt/d/Programs/LEXO/engine/translator.py)
  - добавлен `_postprocess_segment_translation(...)`
  - postprocess включён и для `MockProvider`, и для `NllbProvider`
  - добавлены узкие source-aware cleanup-правила для явных дефектов:
    - `Goodnight -> Спокойной ночи`
    - `Мев -> Мяу`
    - `серыя -> серая`
    - `делает завтрак -> готовит завтрак`
    - `синий футболка -> синюю футболку` для `blue T-shirt`
    - точечная нормализация заголовка `The Sunny Morning`

### Проверки

- `python3 -m py_compile engine/translator.py engine/word_alignment.py` -> OK
- точечные alignment-кейсы:
  - `Tom says. -> [('Tom', 'Том'), ('says', 'говорит')]`
  - `Tom asks. -> [('Tom', 'Том'), ('asks', 'спросил')]`
  - `Tom whispers. -> [('Tom', 'Том'), ('whispers', 'шепчет')]`
  - `Tom thinks. -> [('Tom', 'Том'), ('thinks', 'думает')]`
  - `Anna says hello. -> [('Anna', 'Анна'), ('says', 'говорит'), ('hello', 'привет')]`
- точечные translation cleanup-кейсы:

## Сессия: Добавлены конструкционные didactic rules для `pronoun + be + adjective` и явных possessive

### Причина

- пользователь потребовал уйти от литературного drift, а не латать фразы по одной
- критичные кейсы:
  - `I am great, thank you! -> Я отлично справляюсь, спасибо!`
  - `Luna sleeps on his legs. -> Луна спит на ногах.`
- для учебного режима это недопустимо:
  - предикат `I am + adjective` не должен превращаться в пересказ
  - `his/her/my/...` не должны исчезать из перевода

### Что изменено

- обновлён [engine/didactic_rules.py](/mnt/d/Programs/LEXO/engine/didactic_rules.py)
  - добавлены словари:
    - `PRONOUN_TRANSLATIONS`
    - `BE_TRANSLATIONS`
    - `POSSESSIVE_TRANSLATIONS`
  - `resolve_didactic_translation(...)` теперь покрывает не только точечные фразы, но и конструкционные правила:
    - `pronoun + be + adjective`
    - quoted `pronoun + be + adjective` внутри reporting-клаузы
    - possessive в предлоговых паттернах вроде `on his legs`
    - possessive в прямом дополнении вроде `sees his dog`
  - `_translate_fixed_quote(...)` теперь умеет собирать quote через `_resolve_pronoun_be_rule(...)`, а не только по жёсткому словарю
  - post-edit дополнен защитой от:
    - пропажи possessive в `on his legs`
    - литературного `Я отлично справляюсь` для `I am great`

- обновлён [engine/word_alignment.py](/mnt/d/Programs/LEXO/engine/word_alignment.py)
  - `_repair_possessive_targets(...)` больше не считает `his/her/... -> ''` нормальным финалом
  - добавлен `_find_matching_possessive_target(...)`
  - добавлены таблицы кандидатов:
    - `_target_possessive_candidates_for_source(...)`
    - `_target_noun_candidates_for_possessive_pair(...)`
  - possessive теперь ищется не только по текущему token, но и по паре `possessive + noun`
    - `his -> его`
    - `her -> её`
    - `my -> мой/моих`

### Проверки

- `python3 -m py_compile engine/didactic_rules.py engine/translator.py engine/word_alignment.py` -> OK
- прямые didactic-кейсы:
  - `I am great, thank you! -> Я в порядке, спасибо!`
  - `He is happy. -> Он счастлив.`
  - `She is tired. -> Она уставшая.`
  - `"I am great, thank you!" Anna says. -> "Я в порядке, спасибо!" - говорит Анна.`
  - `Luna sleeps on his legs. -> Луна спит на его ногах.`
  - `Tom sees his dog. -> Том видит его собаку.`
- word alignment:
  - `Luna sleeps on his legs. -> ['Луна', 'спит', 'на', 'его', 'ногах']`
  - `Tom sees his dog. -> ['Том', 'видит', 'его', 'собаку']`

### Ограничение текущей проверки

- активная книга в SQLite всё ещё может показывать старые сегменты:
  - `Я отлично справляюсь, спасибо!`
  - `Луна спит на ногах`
- это старые строки из БД, если книга импортирована до этого среза
- для проверки именно книги нужен ещё один переимпорт или rebuild path для существующих `word_alignments`

## Сессия: Fast-fix падения `slow_native` генерации на Windows console encoding

### Симптом

- при запуске `Slow` generation job уходил в `status=error`
- в `tts_jobs.error_message` зафиксировано:
  - `UnicodeEncodeError` в [engine/tts/kokoro_runner.py](/mnt/d/Programs/LEXO/engine/tts/kokoro_runner.py)

- `cp1251` не умеет кодировать часть phoneme символов
- из-за этого падал уже готовый runner output, хотя audio generation как таковая не была главной причиной ошибки

### Что изменено

- в [engine/tts/kokoro_runner.py](/mnt/d/Programs/LEXO/engine/tts/kokoro_runner.py):
  - `json.dumps(..., ensure_ascii=False)` заменён на:
  - `json.dumps(..., ensure_ascii=True)`

### Проверка

- `python3 -m py_compile engine/tts/kokoro_runner.py engine/tts/tts_provider.py` -> OK

### Примечание

- старый failed job в SQLite остаётся со старой ошибкой
- для подтверждения фикса нужен новый runtime generate/retry через Windows engine
