1. Цель MVP

Сделать живой локальный прототип, который уже можно трогать руками и тестировать

Пользовательский сценарий:

открыть приложение
загрузить .txt книгу
приложение сохраняет книгу локально
приложение обрабатывает текст
пользователь читает текст
пользователь нажимает на слово
приложение показывает перевод и значения
пользователь нажимает озвучку
приложение озвучивает слово или абзац
при повторном запросе всё берётся из кэша

Windows desktop app

рендер текста по абзацам

локальная озвучка абзаца

3. Основная идея архитектуры

Приложение состоит из 2 частей:

3.1 UI

Flutter desktop app

Отвечает за:

окно приложения
экран загрузки книги
экран чтения
карточку слова
кнопку озвучки
показ прогресса
локальные настройки UI
3.2 Локальный движок

Python local engine

Отвечает за:

импорт текста
разбиение книги на абзацы и слова
поиск слов
лемматизацию/нормализацию
перевод слова
поиск значений
генерацию озвучки
кэш
SQLite базу
3.3 Связь

Flutter <-> Python через localhost API

То есть логика не вшивается в интерфейс напрямую

4. Первый пользовательский сценарий
Сценарий №1 — загрузка книги
Пользователь открывает приложение
Видит кнопку Загрузить TXT
Выбирает локальный .txt
Приложение копирует файл в свою папку данных
Движок создаёт запись книги
Движок парсит текст
Текст разбивается на абзацы
Из текста извлекаются слова
Создаётся локальная база книги
Пользователь переходит на экран чтения
Сценарий №2 — чтение
Пользователь видит текст книги
Текст показан по абзацам
Слова кликабельны
Текущая позиция сохраняется автоматически
Сценарий №3 — клик по слову
Пользователь нажимает на слово
UI отправляет запрос движку
Движок:
нормализует слово
ищет перевод в кэше
если не найдено — получает перевод локально
ищет значения/часть речи
сохраняет результат
UI показывает карточку слова
Сценарий №4 — озвучка
Пользователь нажимает кнопку озвучки у слова или абзаца
UI отправляет запрос движку
Движок ищет готовое аудио в кэше
Если нет — генерирует TTS
Сохраняет файл
Возвращает путь к аудио
UI проигрывает файл
5. Экраны MVP
5.1 Экран запуска / Home

Минимальный состав:

название приложения
кнопка Load TXT
кнопка Open current book если книга уже загружена
блок со статусом:
книга не загружена
книга загружена
книга обработана
кнопка Reset current book
Поведение

Если книги нет — показывается только загрузка
Если книга есть — можно сразу открыть чтение

5.2 Экран чтения / Reader

Основной экран MVP

Состав
верхняя панель
название книги
кнопка назад
индикатор прогресса
область текста
абзацы
кликабельные слова
кнопка озвучки рядом с абзацем
нижняя строка статуса
Поведение
текст читается как книга
переносы аккуратные
абзацы отделены
слова по клику открывают карточку
позиция чтения сохраняется автоматически
Важное ограничение

На MVP можно не делать супер-идеальный inline click по каждому слову в RichText, если это будет тормозить разработку
Можно на первом этапе сделать текст через разбивку на токены внутри абзаца и рендерить слово как отдельный интерактивный элемент
Да, это топорно, но для MVP допустимо

5.3 Карточка слова / Word Card

Появляется по клику на слово

Состав
исходное слово
нормальная форма слова
часть речи
основной перевод
дополнительные значения
кнопка Play word
кнопка Save word
кнопка закрыть
Пример данных

Слово: went
Lemma: go
Part of speech: verb
Main meaning: пошёл / пошла / пошли
Other meanings: идти, ехать, направляться

Поведение

Если данные уже были — открыть мгновенно
Если нет — кратко показать загрузку

5.4 Экран словаря / Saved Words

Очень простой экран

Состав
список сохранённых слов
поиск по слову
кнопка озвучки слова
перевод
дата добавления

На MVP без сложной сортировки

8. Что делает движок

Движок — это центральная часть MVP

8.1 Импорт книги

Принимает путь к .txt

Делает:

копию файла
читает текст
очищает мусор
разбивает на абзацы
строит базу
8.2 Токенизация

Для каждого абзаца:

разбить текст на слова и разделители
сохранить слова
подготовить нормализованную форму

На MVP можно:

lowercase
убрать знаки препинания по краям
позже добавить нормальную лемматизацию
8.3 Перевод слова

Принимает слово
Возвращает:

исходное слово
lemma
POS
основной перевод
список значений
8.4 Озвучка

Принимает слово или абзац
Возвращает путь к аудиофайлу

8.5 Кэш

Если перевод или аудио уже были созданы — не считать повторно

9. Что именно должно быть в первом техническом прототипе

Чтобы MVP не размазать, делим на стадии

Стадия A — самый первый рабочий каркас

Должно работать:

приложение открывается
можно загрузить .txt
текст показывается
позиция чтения сохраняется

Без перевода, без озвучки — только скелет

Стадия B — словарная карточка

Добавить:

клик по слову
открытие карточки
возврат данных из движка
кэш слова
Стадия C — TTS

Добавить:

Play word
Play paragraph
кэш аудио
Стадия D — завершение первого MVP

Добавить:

сохранение слова
экран сохранённых слов
reset книги
стабильный запуск из bat

11. API движка для MVP

Минимальный набор методов

GET /health

Проверка, что движок поднят

POST /book/load

Вход:

путь к файлу

Выход:

статус
title
book_id

GET /book/paragraphs

Получить абзацы книги

Вход:

word

Выход:

word

pos
main_translation
meanings

Озвучить абзац

справа

13. Что считать “готовым MVP”

MVP готов, если выполняется этот тест:

Тест готовности
запускаю приложение
загружаю свой .txt
книга сохраняется локально
я читаю текст
нажимаю на слово
вижу перевод и несколько значений
нажимаю озвучку слова
слышу звук
нажимаю озвучку абзаца
слышу абзац
повторный запрос идёт быстрее за счёт кэша
закрываю приложение
запускаю снова
книга и позиция сохранены

Если это есть — MVP состоялся

14. Что не надо усложнять в первом MVP

Вот где легко утонуть:

Не делать сейчас

5 голосов
облачную архитектуру
многокнижную библиотеку
авторизацию
идеальную лемматизацию

Тебе нужен работающий инструмент, а не научная платформа

15. Критерии качества MVP

Чтобы MVP был не мусором, а нормальным первым прототипом, я бы зафиксировал такие требования

Функционально
без ручных танцев запускается
не падает на обычном txt
клик по слову стабильно работает
TTS не зависает интерфейс
UX
текст читается комфортно
карточка слова открывается быстро
есть понятная кнопка play
не нужно 10 действий ради 1 перевода
Технически
данные не теряются после перезапуска
кэш реально работает
UI и движок разделены
проект собирается повторяемо

19. Что получится в итоге

Для тебя это должно выглядеть как одно приложение:

ты запускаешь LEXO.exe
открывается окно
внутри автоматически поднимается local engine
ты работаешь с книгой
все данные лежат локально

То есть для пользователя это одно приложение, хотя внутри два процесса

20. Итоговая формулировка MVP
MVP LEXO v0.1

Локальное Windows-приложение для чтения одной загруженной .txt книги с возможностью нажимать на слова, получать перевод и значения, озвучивать слова и абзацы, кэшировать результаты и сохранять позицию чтения
