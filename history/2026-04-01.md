# LEXO — история за 2026-04-01

## Сессия

План и реализация `MVP5` для multi-level TTS generation, раздельных действий `Generate voice` и `Start`, а также реального прогресса генерации.

## Что изменено

- создан план:
  - `MVP/MVP5_IMPLEMENTATION_PLAN.md`
- создан следующий план:
  - `MVP/MVP6_AUDIO_QUALITY_PLAN.md`
- после пользовательской калибровки speech levels уточнена шкала:
  - оставлены `90`, `120`, `145`, `200`
  - убраны `105` и `170` как недостаточно различимые для практического режима
- по пользовательскому прослушиванию для уровня `90` увеличен `pause_scale`:
  - было `1.4`
  - затем `2.0`
  - затем `2.3`
  - затем `3.0`
  - цель: сделать разрывы между фразами и предложениями заметно более читаемыми
- затем по пользовательскому запросу для уровня `90` включён отдельный speed-эксперимент:
  - вместо стандартного нижнего clamp `0.75`
  - установлен `rate = 0.70`
- Для уровня `90` следующий слуховой эксперимент: `rate` снижен с `0.70` до `0.65`, остальные уровни без изменений.
- Для уровня `90` следующий слуховой эксперимент: `rate` снижен с `0.65` до `0.60`, остальные уровни без изменений.
- В библиотеку добавлено удаление книг: новый backend endpoint `/books/delete`, очистка записи книги из БД, связанных TTS-данных и файлов `data/books/<book_id>` и `data/tts/<book_id>`, а также кнопка `Delete` с подтверждением на экране библиотеки.
- Экран reader переделан под компактный выбор скорости: большие блоки с разными speed/jobs убраны из основного полотна, слева добавлена отдельная боковая TTS-панель с выбором одной скорости, голоса, `Generate`, `Start`, `Stop` и компактным прогрессом только для выбранного уровня.
- Исправлен UI-баг библиотеки: `Open` больше не держит глобальный `_busy` на всё время чтения книги, из-за чего экран выглядел зависшим на `Подождите...`; также выбор job в боковой speed-панели теперь учитывает и `level_id`, и `voice_id`, чтобы не стартовать чужую генерацию при смене голоса.
- Добавлен формальный план `MVP/MVP7_IMPLEMENTATION_PLAN.md` и начата реализация `MVP7`: paragraph cards в reader заменены на reader-first layout, добавлены `TranslationContextBar`, `InteractiveParagraphText` и helper `translation_context_builder.dart`, верхний перевод теперь обновляется по нажатию на английское слово, а позиция чтения по-прежнему сохраняется через `paragraph_index`.
- Верхний translation bar в `MVP7` переделан в одну строку: выбранный перевод теперь остаётся жирным, но встраивается по центру между левым и правым контекстом; также сохранение `/reader/position` больше не спамится на каждый клик по слову внутри одного и того же абзаца.
  - остальные уровни не изменялись
- дополнительно для уровня `90` ужесточена segmentation policy:
  - `target_chunk_chars`: `90`
  - `soft_max_chunk_chars`: `120`
  - `hard_max_chunk_chars`: `150`
  - цель: более короткие и безопасные chunks для медленного режима
- проведён кодовый разбор качества TTS-пайплайна относительно `MVP3`
- зафиксировано, что главная просадка качества сейчас находится в preprocessing:
  - overlap в splitting
  - паузы через `...`
  - упрощённая segmentation policy
- под это создан отдельный следующий этап качества:
  - `MVP6`
- начата реализация `MVP6`:
  - `tts_segmenter.py` переписан с возвратом к char-based policy вместо overlap word-window
  - overlap в splitting убран
  - паузы через `...` убраны из `synthesis_text`
  - введена metadata-пауза `pause_after_ms`
  - playback gap в reader теперь берётся из segment metadata, а не из фиксированного глобального delay
  - `tts_segments` расширены полем `pause_after_ms`
  - добавлена per-speed segmentation policy:
    - `Start 90` режется наиболее осторожно
    - `Transition 120` чуть свободнее
    - `Podcast 145` как базовый естественный режим
    - `Hardcore 200` наиболее крупными chunks
  - в сегментер добавлены boundary guards:
    - запрет слабых окончаний чанка
    - запрет слабых стартов следующего чанка
    - защита от слишком короткого хвоста
    - более умный выбор split point внутри длинной фразы
  - затем learning-group mode для `90` был откатан:
    - мини-группы давали неестественные старты слов и лишний onset перед каждым куском
    - уровень `90` возвращён к обычной sentence/chunk генерации
    - оставлены увеличенные gaps и осторожная segmentation policy
  - затем по запросу пользователя был кратко включён direct word-by-word test для `90`
  - после прослушивания тест сразу откатан как неудачный вариант

  - добавлены миграции схемы для существующей БД

  - длительность cached audio теперь читается из файла, а не оценивается по длине текста
- TTS segmentation переработан под:
  - `paragraph -> sentence -> phrase -> chunk`
  - лимит длины по словам
  - pause scaling через speech profile
- `kokoro_runner.py` теперь принимает `--speed`
- Flutter reader обновлён:
  - `Play` заменён на `Start` на карточках job
  - добавлена отдельная кнопка `Generate voice`

  - добавлен polling `/tts/state`
  - на каждый job выводится отдельная progress bar
  - TTS panel вынесена в отдельный widget, чтобы `reader_screen.dart` не разрастался сверх порога
  - UI-поведение TTS-панели дополнительно подшлифовано:

    - добавлен summary по выбранным levels
    - сырые статусы jobs переведены в более читаемые UI-labels
    - у активных и генерирующихся jobs добавлены status chips
  - у каждой speed/job карточки добавлена кнопка `Stop`:
