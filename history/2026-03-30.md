# LEXO — история за 2026-03-30

## Сессия

Фикс запуска MVP через bat-файл без зависимости от `PATH`

## Что изменено

- в `run_lexo_mvp.bat` задан явный путь:
  - `C:\src\flutter\bin\flutter.bat`
- вызовы Flutter переведены на форму с кавычками:
  - `call "%FLUTTER_CMD%" pub get --directory app`
  - запуск `flutter run -d windows` перенесён в папку `app` через `pushd app`
- убран неподдерживаемый в текущем окружении флаг:
  - `--project-dir app`
- обновлён `app/windows/flutter/CMakeLists.txt` под текущий шаблон Flutter Windows
- добавлены отсутствовавшие определения:
  - `FLUTTER_LIBRARY`
  - `FLUTTER_ICU_DATA_FILE`
  - `PROJECT_BUILD_DIR`
  - `AOT_LIBRARY`
- обновлена логика сборки wrapper и `tool_backend.bat` для совместимости с Flutter 3.35
- синхронизированы файлы `app/windows/runner/*` с текущим Flutter Windows runner template
- добавлены недостающие объявления и функции:
  - `Utf8FromUtf16`
  - DPI/theme support в `win32_window.*`
  - `generated_plugin_registrant.cc` и `dwmapi.lib` в `runner/CMakeLists.txt`
- устранён конфликт имён `WindowClassRegistrar` в `win32_window.cpp/.h`
- синхронизирована верхняя install-конфигурация `app/windows/CMakeLists.txt` с текущим Flutter template
- добавлено диагностическое логирование действий UI и HTTP-запросов
- добавлено логирование запросов/ответов в Python engine
- устранены необработанные исключения на кнопках TTS в Flutter UI
- добавлена диагностика чтения POST-body в `engine/main.py` через заголовки и raw body
- добавлена поддержка `Transfer-Encoding: chunked` для Flutter POST-запросов в engine

## Зачем

- у пользователя Flutter установлен, но не прописан в `PATH`
- из-за этого `run_lexo_mvp.bat` не находил `flutter`
- bat-файл теперь использует Flutter напрямую и не зависит от системной переменной `PATH`
- дополнительно устранён сбой запуска UI из-за несовместимости `flutter run` с флагом `--project-dir`
- дополнительно устранён сбой CMake из-за устаревшего `app/windows/flutter/CMakeLists.txt`
- дополнительно устранены compile-ошибки Windows runner из-за устаревших `runner`-файлов
- дополнительно устранена двусмысленность класса registrar и ошибка доступа к `WndProc`
- дополнительно устранён сбой шага `INSTALL`, который пытался ставить сборку в `C:\Program Files\lexo_app`
- дополнительно добавлена видимость пользовательских действий и API-ошибок в консоли
- дополнительно добавлена диагностика пустых POST-payload для Flutter -> engine
- дополнительно устранён runtime-сбой пустых payload при `chunked` POST

## Ожидаемый результат

- `run_lexo_mvp.bat` должен проходить проверку Flutter
- при первом запуске должен выполнять `pub get`
- затем должен поднимать Python engine и запускать Flutter UI
- запуск Flutter должен выполняться из каталога `app`, без `--project-dir`
- Windows runner должен проходить стадию генерации build files без ошибки `Flutter library not found`
- Windows runner должен собираться без ошибок `C2440`, `Utf8FromUtf16`, `EnableFullDpiSupportIfAvailable`
- `flutter run` должен собирать bundle рядом с `.exe`, без установки в `Program Files`
- в консоли должны быть видны клики, запросы UI и ответы/ошибки engine
- в консоли должны быть видны заголовки и raw body для POST-запросов
- POST-запросы из Flutter должны доходить в engine с реальным JSON payload
