# MVP15 — Cards List And Swipe Review Plan

## 1. Цель этапа

Построить поверх уже реализованного `MVP14` полноценный минимальный цикл обучения карточек:

- создать карточку прямо из reader через `long press`
- сохранять карточки локально
- показывать отдельный экран `Cards`
- запускать простой review-режим
- учить карточки через `swipe left / right`
- показывать простой прогресс без тяжёлого SRS

Итог этапа:

- `Reader` остаётся точкой входа
- `Detail Sheet` остаётся местом понимания и добавления карточки
- появляется отдельный `Cards` storage layer
- появляется экран списка карточек
- появляется экран повторения
- появляется простая progress model `0..3`

## 2. Границы этапа

Этап строится поверх уже существующего:

- `tap` быстрый перевод
- `long press -> Detail Sheet`
- `lexical_enrichment`
- detail-level units:
  - `LEXICAL`
  - `PHRASE`
  - `GRAMMAR`
- сохранение `LEXICAL / PHRASE`

Этап не должен:

- тянуть внешний словарь
- вводить новую языковую модель
- строить сложный spaced repetition
- добавлять облачную синхронизацию

## 3. Главный продуктовый смысл

Новая фича должна замкнуть цикл:

`Reader -> Long press -> Popup -> Add card -> Cards list -> Review by swipe -> Progress`

То есть:

- reader помогает найти языковую единицу
- popup помогает её понять
- `Add card` отправляет её в локальную базу карточек
- `Cards` показывает всё сохранённое
- `Review` превращает сохранённое в повторение

## 4. Основной пользовательский сценарий

### 4.1 Во время чтения

Пользователь читает книгу.

- обычный `tap` -> быстрый перевод
- `long press` -> `Detail Sheet`

Из `Detail Sheet` пользователь может:

- сохранить `LEXICAL` card
- сохранить `PHRASE` card
- при желании сохранить `GRAMMAR` отдельно

По умолчанию grammar не должна массово попадать в основной review deck.

### 4.2 После добавления

Сохранённая карточка попадает в локальную БД карточек.

### 4.3 На экране Cards

Пользователь:

- открывает список сохранённых карточек
- фильтрует их
- видит статус и прогресс
- запускает review session

### 4.4 В review-режиме

Пользователь:

- смотрит одну карточку на экран
- открывает ответ
- свайпает влево или вправо
- двигает карточку по простому прогрессу

## 5. Что считается карточкой

Карточка — это одна языковая единица.

Типы карточек:

- `LEXICAL`
- `PHRASE`
- `GRAMMAR`

### 5.1 Lexical

Примеры:

- `sun`
- `run`
- `house`

### 5.2 Phrase

Примеры:

- `take off`
- `in front of`
- `out of`

### 5.3 Grammar

Примеры:

- `the`
- `a`
- `of`
- `to`

## 6. Как карточка создаётся из reader

### 6.1 Обычная группа

Пример:

- `the sun`

Reader selection остаётся как сейчас.

`Detail Sheet` разбирает группу на units:

- `the`
- `sun`

Popup показывает:

- grammar unit: `the`
- lexical unit: `sun`

Кнопка `Add` по умолчанию сохраняет:

- `sun`

### 6.2 Phrase / MWE

Пример:

- `in front of`

Система распознаёт это как одну `PHRASE` unit.

Popup показывает:

- одну phrase-card: `in front of`

Кнопка `Add` сохраняет:

- `in front of`

### 6.3 Phrasal verb

Пример:

- `took off`

Система распознаёт:

- surface form: `took off`
- head/lemma: `take off`

Popup показывает:

- одну phrase-card: `take off`

## 7. Что показывается в popup при long press

## 7.1 Lexical

Пример:

```text
sun
солнце

the sun -> солнце
```

Если у слова другая форма, добавить короткую строку:

```text
ran -> прошедшее
```

или

```text
running -> форма с -ing
```

Без длинной теории и без таблиц.

## 7.2 Grammar

Пример:

```text
the
конкретный / тот самый
```

Коротко, одной строкой.

Для grammar-слов, у которых есть прямой смысл, допускается показывать его напрямую:

- `he -> он`
- `is -> есть`

## 7.3 Phrase

Пример:

```text
in front of
перед

in front of the house -> перед домом
```

Допустима короткая строка `как понимать`, только если реально полезно.

## 8. Внешний вид popup

При `long press` открывается нижняя панель или modal sheet.

Внутри:

- если в группе phrase -> одна карточка / один unit block
- если не phrase -> несколько unit blocks подряд

Порядок для MVP:

- сначала `LEXICAL / PHRASE`
- затем `GRAMMAR`

То есть смысловые единицы должны идти выше grammar.

## 9. Что сохраняется в локальную базу карточек

Нужен отдельный storage layer для карточек, а не только `source_words`.

### 9.1 Новая таблица

- `saved_cards`

### 9.2 Минимальные поля

- `id`
- `card_type`
- `head_text`
- `surface_text`
- `lemma`
- `translation`
- `example_text`
- `example_translation`
- `pos`
- `grammar_label`
- `source_book_id`
- `source_paragraph_id`
- `source_segment_id`
- `created_at`
- `status`
- `progress_score`
- `review_count`
- `last_reviewed_at`

### 9.3 Смысл полей

`card_type`

- `lexical`
- `phrase`
- `grammar`

`head_text`

- основная форма карточки
- для phrase-card это phrase head

`surface_text`

- форма, в которой карточка встретилась в книге

`translation`

- короткий перевод карточки

`example_text`

- пример из книги

`example_translation`

- перевод примера из контекста

`grammar_label`

- короткая grammar / form / helper-метка

## 10. Что сохраняется по умолчанию

По умолчанию:

- сохраняем `LEXICAL`
- сохраняем `PHRASE`

Не сохраняем массово:

- `GRAMMAR`

Grammar-card:

- можно открыть
- можно при желании сохранить отдельно
- но по умолчанию не включать в основной review deck

## 11. Статусы карточки

Для MVP нужны 4 статуса:

- `new`
- `learning`
- `known`
- `mastered`

### 11.1 Логика

`new`

- карточка только что добавлена

`learning`

- пользователь уже начал её свайпать

`known`

- знает более-менее

`mastered`

- уверенно знает

## 12. Шкала прогресса

Не нужен Anki-уровень.

Достаточно:

- `0`
- `1`
- `2`
- `3`

Или визуально:

- `0/3`
- `1/3`
- `2/3`
- `3/3`

## 13. Экран Cards

Нужен отдельный экран / вкладка с двумя режимами.

### 13.1 Режим списка

Показывает все сохранённые карточки.

Для каждой строки:

- слово / фраза
- перевод
- тип
- статус
- маленький прогресс

Пример:

- `sun — солнце — lexical — 1/3`
- `take off — снять — phrase — 0/3`

### 13.2 Фильтры

Для MVP достаточно:

- `all`
- `new`
- `learning`
- `known`

### 13.3 Режим review

Из списка пользователь запускает review session.

## 14. Экран Review

Показывается одна карточка на экран.

### 14.1 Лицевая сторона

Показывает:

- `head_text`
- при необходимости короткий example context

Примеры:

- `run`
- `take off`
- `the`

### 14.2 Reveal

После тапа или кнопки `Показать` открывается ответ:

- перевод
- пример из текста
- short form label, если есть

Пример:

```text
run
бежать

ran -> прошедшее

He ran home
Он побежал домой
```

## 15. Swipe logic

Для MVP достаточно двух свайпов.

### Swipe left

- плохо / не знаю

### Swipe right

- знаю / хорошо

### Поведение

Если `left`:

- карточка остаётся в `learning`
- прогресс не растёт или падает на 1

Если `right`:

- прогресс растёт на 1
- карточка двигается к `known` / `mastered`

## 16. Простая progress logic

### `new`

Первый `right`:

- `learning`
- `1/3`

### `learning`

Ещё один `right`:

- `known`
- `2/3`

### `known`

Ещё один `right`:

- `mastered`
- `3/3`

### `left`

Для MVP лучше:

- не откатывать слишком жёстко
- либо оставить на месте
- либо уменьшить на `1`

Предпочтительная MVP-логика:

- `left` снижает не более чем на один шаг

## 17. Поведение grammar-card в review

Grammar-card не должна вести себя как массовая lexical-card.

Для MVP:

- grammar-card можно открыть и понять
- можно сохранить отдельно
- но по умолчанию она не должна попадать в основной review deck

То есть:

- grammar = справочный слой
- lexical / phrase = основа массового повторения

## 18. Поведение phrase-card

Phrase-card ведёт себя как обычная review-card.

Пример:

```text
in front of
перед
```

Она:

- сохраняется
- повторяется
- свайпается
- имеет прогресс

## 19. Внешний вид карточки в review

### Lexical

Лицевая:

- `sun`

Обратная:

- `солнце`
- `the sun -> солнце`

### Grammar

Лицевая:

- `the`

Обратная:

- `конкретный / тот самый`

### Phrase

Лицевая:

- `in front of`

Обратная:

- `перед`
- `in front of the house -> перед домом`

## 20. Что показывать, а что не показывать

Показывать:

- слово / фразу
- перевод
- короткий пример из книги
- короткую form label:
  - `ran -> прошедшее`
  - `running -> форма с -ing`

Не показывать:

- длинные определения
- энциклопедические объяснения
- 5-10 примеров
- длинную грамматическую теорию
- отдельный словарь значений

## 21. Как карточка создаётся технически

Источник данных:

- текущий translation pipeline
- `lexical_enrichment`
- пример из сегмента
- `translation_focus`
- group translation

То есть:

- карточка собирается из уже существующей обработки книги
- плюс enrichment
- плюс новый storage layer `saved_cards`

## 22. Что добавить в backend

Нужен отдельный storage для карточек.

### 22.1 Минимально

- новая таблица `saved_cards`

### 22.2 Новые действия

- `list cards`

- `start review session`

## 24. MVP review flow

Пользователь открывает `Cards`.

Видит:

- сколько `new`
- сколько `learning`
- сколько `known`

Жмёт `Начать повторение`.

Открывается серия карточек:

- сначала `new`
- потом `learning`

На карточке:

- headword / phrase
- `Показать`
- `swipe left / right`

После серии:

- мини-результат

Пример:

- `12 карточек просмотрено`
- `7 продвинулись`
- `3 остались сложными`
- `2 уже закреплены`

## 25. Что считается готовым MVP

Этап считается готовым, если:

- `long press` на тексте открывает popup
- popup показывает units корректно
- phrase не распадаются
- lexical и grammar различаются
- можно добавить `LEXICAL / PHRASE` в базу карточек
- есть экран списка карточек
- есть review screen
- есть `swipe left / right`
- есть простой progress `0..3`
- всё работает офлайн

## 26. Что осознанно не входит

Не включаем:

- внешний словарь
- сложный spaced repetition
- ручное редактирование карточки
- несколько переводов слова
- теги, папки, кастомные колоды
- отдельную grammar-программу
- audio на каждую карточку
- синхронизацию с сервером
