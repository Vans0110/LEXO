# 2026-04-08

## Сессия: mobile reader переведён на strict offline runtime, host оставлен только для sync

### Что сделано

- обновлён `app/lib/src/ui/mobile/screens/mobile_reader_screen.dart`
- mobile reader больше не делает неявных host-вызовов во время чтения:
  - при открытии книги больше не тянутся remote `tts profiles / levels / state`
  - сохранение позиции чтения больше не отправляется в host
  - `detail sheet` больше не запрашивается с host и всегда строится локально из package payload
  - playback больше не делает remote fallback download/control
- mobile playback теперь работает только от локально синхронизированного audio:
  - если локального audio нет, reader показывает явное сообщение про необходимость `Синхронизации`
  - stop/reset состояния теперь тоже выполняется локально
- TTS generation/update внутри mobile reader отключены как runtime-сценарий:
  - reader сообщает, что такие действия выполняются только через `Синхронизацию с host`
- обновлён `scripts/run_lexo_mobile_android_inner.bat`
  - Android mobile launcher больше не поднимает Python host автоматически
  - mobile UI и LAN host теперь запускаются отдельно
  - для host-based sync пользователь должен отдельно запускать `run_lexo_engine_lan.bat`
  - из Android launcher также убран автоматический `adb reverse`
  - Flutter mobile app больше не стартует с принудительным `LEXO_BASE_URL=http://127.0.0.1:8765`
  - итог: launcher больше не создаёт автоматическую связь mobile -> host даже на emulator
- обновлён `app/lib/src/mobile/mobile_settings_repository.dart`
  - для mobile settings задан дефолтный `Host URL`:
    - `http://192.168.1.50:8765`
  - этот адрес подставляется сразу, даже если локальный settings-файл ещё не создан
  - пустой `host_url` в сохранённом JSON тоже нормализуется обратно в этот дефолт
  - дефолт теперь можно переопределять через `LEXO_DEFAULT_MOBILE_HOST_URL`
- обновлены `app/lib/src/api/api_client.dart`, `app/lib/src/mobile/mobile_sync_debug_logger.dart`, `app/lib/src/ui/mobile/screens/mobile_shell_screen.dart`
  - перед `Синхронизацией` добавлен быстрый `host` probe через `/health`
  - если host недоступен, sync теперь прерывается сразу с явной ошибкой вместо долгого ожидания socket timeout
  - remote debug-log для sync переведён в fail-fast режим:
    - использует короткий timeout
    - после первой ошибки отключается на текущую сессию

  - это устраняет падение sync на:
    - `UNIQUE constraint failed: index 'idx_saved_cards_unique'`
  - tombstone/live merge для такой semantic-карточки теперь использует `updated_at`, вместо слепого `INSERT`
- дополнительно облегчён ответ `POST /mobile/sync/full`
  - backend больше не возвращает весь `merged_cards` payload
  - вместо этого отдаётся компактный `merged_cards_count`
  - mobile client при таком ответе отдельно делает `GET /cards` и уже этим pull-списком обновляет локальные cards
  - это сделано как transport-fix для Android emulator, где большой JSON от `sync/full` рвался на чтении
- добавлен новый протокол загрузки mobile book package:
  - `GET /mobile/books/package-manifest`

- mobile client (`Sync` и `Update`) переведён на загрузку книги через `manifest + parts`

  - фиксированный chunk по 12 абзацев оказался всё ещё слишком большим

- переработан Android launcher под два явных режима эмулятора:

  - телефонный сценарий по LAN IP при этом не меняется

### Зачем

- это устраняет ложный hybrid-runtime, когда mobile книга уже локальная, но reader всё равно ждал сетевой провал host перед fallback
- итоговая цель:
  - `Sync` и явные host-действия остаются в Settings/Library
  - всё остальное внутри mobile reader работает автономно
- дополнительно это устраняет смешанный Android launcher, где mobile UI раньше автоматически стартовал host вместо явного разделения ролей
- emulator теперь ближе к поведению реального телефона:
  - host запускается отдельно
  - адрес host задаётся вручную через mobile `Host URL`
  - связь выполняется только по явному действию `Синхронизировать`
